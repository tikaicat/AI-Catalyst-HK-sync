# ACH Web Artifacts Builder Skill - Trigger Prompt

## 🎯 Skill Purpose
建立和優化Web應用程序、組件和互動式內容

## 📌 Trigger Scenarios

### When to Use
1. **Web應用程序** - 需要構建交互式Web應用
2. **UI組件** - 創建可重用的設計組件
3. **互動式演示** - 建立產品展示或概念驗證
4. **Web優化** - 改進現有Web內容的性能和UX

## 💬 Sample Trigger Prompts

### Scenario 1: Build Interactive Web App
```
使用 ach-web-artifacts-builder：
請為 [功能] 構建Web應用程序：
- 用戶場景：[描述]
- 主要功能：[功能列表]
- 技術要求：[技術棧]
- 設計參考：[樣式/品牌]
- 部署目標：[部署位置]
```

### Scenario 2: Create UI Component Library
```
使用 ach-web-artifacts-builder：
為我們的 [項目/品牌] 創建UI組件庫：
- 組件列表：[組件]
- 設計系統：[設計標準]
- 樣式變體：[變體]
- 文檔和示例
- 可訪問性要求
```

### Scenario 3: Interactive Demo/POC
```
使用 ach-web-artifacts-builder：
請為 [概念/產品] 創建交互式演示：
- 演示流程：[步驟]
- 關鍵交互：[交互]
- 視覺效果：[效果]
- 數據模擬：[數據]
- 響應式設計：[設備]
```

## 📋 Output Format

✅ 應該包含：
- HTML/CSS/JavaScript 代碼
- 組件文檔
- 使用示例
- 浏覽器兼容性信息
- 性能優化建議
- 可訪問性檢查清單
- 部署說明

## 💡 Pro Tips
- 使用現代 Web 技術（HTML5, CSS3, ES6+）
- 確保響應式設計（移動優先）
- 包含暗模式支持
- 提供清晰的可訪問性標籤
- 優化首屏加載時間
