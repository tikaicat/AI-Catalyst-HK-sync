# ACH Doc Coauthoring Skill - Trigger Prompt

## 🎯 Skill Purpose
與團隊成員協作編輯和完善文檔

## 📌 Trigger Scenarios

### When to Use
1. **多人編輯** - 需要與團隊成員共同編輯文檔
2. **審查流程** - 文檔需要經過審查和批准
3. **版本管理** - 需要管理文檔的多個版本
4. **內容統一** - 確保多個作者的內容風格一致

## 💬 Sample Trigger Prompts

### Scenario 1: Team Document Collaboration
```
使用 ach-doc-coauthoring：
我需要與 [團隊成員] 協作編輯 [文檔名稱]：
- 負責部分：[描述]
- 協作者負責部分：[描述]
- 截止日期：[日期]
- 審查準則：[標準]
```

### Scenario 2: Document Review Process
```
使用 ach-doc-coauthoring：
請組織 [文檔] 的審查流程：
- 第一輪：[審查者1] 審查 [部分]
- 第二輪：[審查者2] 審查整體一致性
- 最終：[批准者] 簽署
- 反饋模式：[內聯注釋/追蹤更改]
```

### Scenario 3: Content Style Consistency
```
使用 ach-doc-coauthoring：
確保文檔的內容風格一致：
- 應用品牌語言指南
- 統一術語表
- 檢查格式和結構
- 修復不一致之處
```

## 📋 Output Format

✅ 應該包含：
- 清晰的協作結構
- 明確的責任分配
- 評論和建議
- 版本歷史
- 最終批准標記

- 使用清晰的評論格式（[評論者] - 評論內容）
- 為大型文檔設置分章節的審查時間表
- 保持所有改動的詳細記錄
- 定期同步和狀態更新
